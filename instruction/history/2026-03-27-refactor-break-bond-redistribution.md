# Summary (since last /compact)

Bond breaking in 3D no longer uses the VSEPR preview pipeline (`TryPreviewVseprSlotsForBreakBond`, homolytic slot symmetricizer, or edit-mode heavy-framework slot preservation keyed off H destroy). Break paths always pass π angle and ref world direction into redistribution (not only when π count drops to zero). `CoAnimateBreakBondRedistribution` snapshots the molecule, runs a single full `RedistributeOrbitals` + `OrientEmptyNonbondedOrbitalsPerpendicularToFramework` pass to define final non-bond locals, restores the pre-phase snapshot, then animates only toward frozen non-bond targets and applies those frozen locals at the end (no second redistribute/orient that could teleport empties). Bond→slot animation is suppressed for 0e break orbitals and for π-staged guides so intermediate targets do not fight the final geometry.

`AtomFunction` adds `NonBondEmptyOrbitalCount`, helpers to snapshot empty/non-bond orbital targets, `TryGetPerpendicularEmptyTargetForGuide`, lone→empty electron transfer ordering comments, and rewrites `RedistributeOrbitals3D` to compute targets via `GetRedistributeTargets3D`, `ApplyRedistributeTargets`, perpendicular placement for 0e guide orbitals, and optional `[RedistributeOrbitals3D-diag]` pairwise-angle logs. `GetRedistributeTargets3D` disables several break-specific early exits (carbocation / two-sigma trigonal / rigid σ cleavage) in favor of the main VSEPR path with clarified guide pinning. `OrientEmptyNonbondedOrbitalsPerpendicularToFramework` relaxes the π=0 single-empty skip so 0e guides still get the final perpendicular snap unless the caller explicitly pins that orbital.

`EditModeManager` removes redundant post–`FormSigmaBondInstant` `RedistributeOrbitals` calls in 3D (keeps H snap). `ElectronOrbitalFunction` stops calling `TryRedistributeOrbitalsAfterBondChange` after animated bond formation to avoid lone-pair drift after H snapping. Debug defaults: `DebugLogBondBreakEmptyTeleport` and `DebugLogBreakBondMotionSources` set true for triage.
