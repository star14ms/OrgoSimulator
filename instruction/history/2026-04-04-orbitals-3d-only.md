# Orbitals: remove 2D geometry, standardize on 3D

Since last `/compact` (ref was pending): the working tree drops orthographic / XY work-plane orbital behavior and keeps a single 3D pipeline. `OrbitalAngleUtility` now treats directions as full 3D and exposes `UseFull3DOrbitalGeometry => true` for any remaining references. `AtomFunction` removes the old 2D redistribute path (`RedistributeOrbitals` → `RedistributeOrbitals3D` only), simplifies `GetRedistributeTargets`, `ApplyRedistributeTargets`, slot/orbital creation, selection highlight, and permutation cost helpers to 3D-only math. `CovalentBond` bond-break always uses the lerp + `CoLerpBondBreakRedistribution` path and 3D ex-bond lobe poses. `EditModeManager` and `MoleculeBuilder` use 3D VSEPR / hydrogen saturation / FG basis without flattening `z` or planar-only helpers; cycloalkane anchoring matches the previous perspective rule (explicit attach to selection). `ElectronOrbitalFunction` forces 3D presentation and removes XY-only drag/orbit helpers. `.gitignore` ignores `*.ndjson` and fixes a missing newline at EOF.
