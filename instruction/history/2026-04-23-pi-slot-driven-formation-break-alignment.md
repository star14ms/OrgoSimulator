Pi bond handling was refactored to use `piPOrbitalDirectionSlots` as the primary source of truth across formation, redistribution, and break cleanup. `AtomFunction` now stores partner/line-indexed pi-axis slots with helpers for prebond and committed-row lookup, and `SigmaBondFormation` computes shared prebond +z from the latest stored row (falling back to OP geometry) so OP rotation and redistribution use the same axis authority. `ElectronOrbitalFunction` phase-1 cylinder targets were aligned to prebond slot +z instead of perspective lobe inference to prevent animation/final-direction divergence. `CovalentBond` now promotes prebond direction into committed row slots on pi bond creation and removes broken-row slots with a robust effective-row fallback when stored line indices drift. Temporary debug-session instrumentation was then removed after verification, while production slot-management and alignment fixes were kept.
